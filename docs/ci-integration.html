CI Integration

Automatically deploy a preview environment for every merge request. Mayfly provides a purpose-built Docker image that makes CI setup a single job.

GitLab CI

Add the following job to your .gitlab-ci.yml. The ghcr.io/mikestreety/ddev-hosted image contains ddev and the Mayfly CLI pre-installed.

.gitlab-ci.yml
deploy:preview:
  stage: deploy
  image: ghcr.io/mikestreety/ddev-hosted:latest
  script:
    - preview-deploy

The job will deploy on every push. To limit it to merge requests only, add a rules block:

.gitlab-ci.yml
deploy:preview:
  stage: deploy
  image: ghcr.io/mikestreety/ddev-hosted:latest
  rules:
    - if: $CI_MERGE_REQUEST_ID
  script:
    - preview-deploy

GitHub Actions

For GitHub Actions, use the same Docker image as your job container:

.github/workflows/preview.yml
name: Deploy Preview
on:
  pull_request:
jobs:
  preview:
    runs-on: ubuntu-latest
    container:
      image: ghcr.io/mikestreety/ddev-hosted:latest
    steps:
      - uses: actions/checkout@v4
      - run: preview-deploy

SSH key in CI

Your CI runner needs access to your Mayfly SSH private key. Store it as a masked CI/CD variable and pass it through to the job:

GitLab CI
before_script:
  - eval $(ssh-agent -s)
  - echo "$MAYFLY_SSH_KEY" | ssh-add -
Security: Store your private key as a masked, protected variable. Never commit it to your repository.

Environment URLs

After a successful deploy, preview-deploy prints the environment URL to stdout. You can capture it and post it as a merge request comment using your CI platform's API.
