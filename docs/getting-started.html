Getting Started

Getting Started with Mayfly

Mayfly gives you on-demand preview environments powered by ddev. Deploy a fresh copy of your site for every merge request, share the URL with your team, and let it disappear when you're done.

Prerequisites

  • ddev installed and configured on your local machine
  • An SSH key pair — Mayfly uses your public key for authentication
  • A Mayfly account (free during beta — join via Slack)

Installation

Run the installer with a single command. It will set up the mayfly CLI and register your SSH key with your account.

bash
$ bash <(curl -fsSL get.mayfly.live/preview.sh)

The installer will prompt you to paste your public SSH key if one has not been registered yet.

Deploying your first environment

Navigate to the root of any ddev-powered project and run:

bash
$ preview-deploy

Mayfly will spin up the environment under your account subdomain and print a shareable URL.

Tip: Each environment is named after your project and branch automatically. Run the same command on a different branch and you'll get a second, independent environment — both live at the same time.

Tearing down

Environments are ephemeral by design, but you can destroy one manually:

bash
$ preview-destroy
